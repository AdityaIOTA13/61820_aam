# 61820_aam — 360° Courtyard Scan for Incremental Coverage Mapping

A research dataset and processing pipeline that maps a GPS-traced 360° walk through a courtyard to a georeferenced floorplan, enabling depth-map-based coverage reconstruction.

---

## Getting Started

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Extract frames from the 360° video at 2 fps → frames/
python extract_frames.py

# 3. Match each frame to its real-world XY position on the floorplan → frame_positions.json
python sync_gps_video.py

# 4. Run depth estimation on every frame → depth_maps/
python estimate_depth.py
```

## Key Output: `frame_positions.json`

`frame_positions.json` is the bridge between the video and the floorplan. It maps every extracted frame to a real-world position and timestamp:

```json
{
  "frame_00000_0.000s.jpg": {
    "x_px": 373.5,
    "y_px": 475.5,
    "x_meters": 17.873,
    "y_meters": 16.455,
    "timestamp_sec": 0.0
  },
  ...
}
```

With this file you can:
- **Project depth maps onto the floorplan** — for each frame, use its `(x_meters, y_meters)` as the camera origin and cast the corresponding depth map into top-down floorplan space
- **Assign map age** — `timestamp_sec` lets you colour-code each coverage patch by when it was observed, enabling temporal coverage analysis
- **Build an incremental coverage heatmap** — accumulate projected patches across all frames to show what fraction of the courtyard was visible at each depth threshold

## Pipeline Overview

```
360° video (R2)
      │
      ▼
extract_frames.py  →  frames/  (2 fps JPEGs)
      │
      ├──► sync_gps_video.py  →  frame_positions.json  (XY + timestamp per frame)
      │
      └──► estimate_depth.py  →  depth_maps/  (grayscale PNGs)
                                        │
                                        ▼
                           [next] project_coverage.py  →  coverage heatmap on floorplan
```
