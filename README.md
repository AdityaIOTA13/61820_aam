# 360° Courtyard Scan — Incremental Coverage Mapping Dataset

This repository contains raw and processed data from a 360° indoor/outdoor mapping session conducted in a courtyard environment. It is part of an **incremental coverage mapping research project** that aims to reconstruct spatial coverage from egocentric 360° video using depth estimation and path-aware compositing.

---

## Project Overview

The goal is to generate a full **coverage map** of the courtyard by:
1. Walking a calibrated GPS-traced path through the space while recording equirectangular 360° video
2. Estimating depth from each video frame (monocular depth estimation)
3. Projecting visible surfaces onto the floorplan at each recorded position
4. Incrementally compositing coverage patches to build a complete spatial map

## GPS Walk Path (`data/svg_path.svg`)

The SVG path is a **calibrated GPS walk trace** derived from a GNSS recording taken during the 360° video capture session. The raw GPS coordinates were:

1. Projected into the floorplan pixel coordinate space using known ground-control reference points
2. Smoothed and resampled to remove GPS jitter
3. Exported as an SVG `<path>` element with the same `1400×819` viewBox as the floorplan

The path has **14 waypoints** and traces a U-shaped route along the corridor and through the center of the courtyard. Waypoints are numbered sequentially from start (green) to end (red) in `floorplan_walkpath.png`.

### Re-uploading / Credentials

Copy `.env.example` to `.env` and fill in your Cloudflare R2 credentials, then run:

```bash
source .env
aws s3 cp <local_video.mp4> s3://$R2_BUCKET/courtyard_360/<filename>.mp4 \
  --endpoint-url "https://$R2_ACCOUNT_ID.r2.cloudflarestorage.com" \
  --region auto \
  --content-type "video/mp4"
```

## Next Steps

- [ ] **Depth estimation** — run a monocular depth model (e.g., [Depth Anything V2](https://github.com/DepthAnything/Depth-Anything-V2)) on equirectangular frames, handling the spherical projection distortion
- [ ] **Frame ↔ path alignment** — sync video timestamps to GPS waypoints to assign each frame a spatial position on the floorplan
- [ ] **Coverage projection** — for each frame, project the estimated depth map into top-down floorplan space, masking by visible angle
- [ ] **Incremental coverage map** — accumulate projected patches across all frames to build a complete colored coverage heatmap overlaid on the floorplan
- [ ] **Coverage metric** — compute fraction of courtyard area observed at each depth threshold
